# 16.4 用户访谈技巧 🟢

> **阅读完本节后，你将会收获：**
> - 掌握 The Mom Test 的核心原则
> - 学会区分好问题和坏问题
> - 了解用户访谈的完整流程
> - 掌握分析和应用访谈结果的方法

> 有些反馈很模糊，比如有人说"不好用"。这时候需要做用户访谈，找到真正的问题。

---

## The Mom Test 原则

The Mom Test 是一套用户访谈方法，核心是通过好问题获取真实数据。

### 三条核心原则

| 原则 | 说明 |
|------|------|
| **谈论过去而非未来** | 过去的行为是真实的，未来的意愿可能不兑现 |
| **谈论具体而非笼统** | 细节能揭示真实问题，笼统回答没有价值 |
| **少问多听** | 让用户多说，从细节中发现真实需求 |

::: tip 为什么叫 The Mom Test

即使问你的妈妈，她也无法对这些好问题撒谎，因为问题关注的是过去的具体事实，而非未来的承诺。

:::

## 好问题 vs 坏问题

### 坏问题类型

| 坏问题 | 为什么不好 | 好问题替代 |
|--------|-----------|-----------|
| "你会用这个功能吗？" | 未来意愿不可靠 | "你上次怎么解决类似问题的？" |
| "这个功能重要吗？" | 礼貌性肯定 | "你什么时候需要用到这个？" |
| "你觉得这个设计怎么样？" | 可能只是客套 | "这个设计让你哪里困惑？" |
| "你想要什么功能？" | 用户不是产品经理 | "你最近遇到什么困难？" |

### 好问题类型

| 好问题 | 为什么有效 |
|--------|-----------|
| "你上次遇到什么问题？" | 基于真实经历 |
| "你当时怎么处理的？" | 了解当前解决方案 |
| "这个问题多久出现一次？" | 判断重要性 |
| "能给我演示一下吗？" | 观察而非询问 |

## 访谈流程

一个完整的用户访谈包含多个阶段。

### 访谈前准备

| 准备项 | 说明 |
|------|------|
| **明确目的** | 想要验证什么假设 |
| **寻找参与者** | 符合目标用户画像 |
| **准备问题** | 列出要问的核心问题 |
| **安排环境** | 安静、无干扰 |
| **准备记录** | 录音或笔记工具 |

### VFWPA 会议邀请格式

使用 VFWPA 格式发送邀请：

| 部分 | 内容 | 示例 |
|------|------|------|
| **V** Vision | 会议愿景 | 了解用户如何使用产品 |
| **F** Finish | 预期产出 | 发现 3 个可改进点 |
| **W** Why | 为什么需要 | 当前反馈太模糊 |
| **P** Plan | 会议计划 | 30 分钟，用户演示使用过程 |
| **A** Agenda | 议程 | 开场 → 演示 → 提问 → 总结 |

### 访谈中

| 阶段 | 要点 |
|------|------|
| **开场** | 说明目的，强调没有"正确答案" |
| **观察** | 让用户演示，不要打断 |
| **提问** | 用好问题深入挖掘 |
| **记录** | 记录原话，不要只记总结 |
| **感谢** | 感谢用户的时间和坦诚 |

### 访谈后

| 步骤 | 说明 |
|------|------|
| **整理记录** | 24 小时内整理，记忆会淡化 |
| **提炼洞察** | 找出反复出现的主题 |
| **分享团队** | 让团队了解真实用户 |
| **决定行动** | 根据发现调整计划 |

## 访谈提问技巧

### 开放式问题

| 类型 | 示例 |
|------|------|
| **背景** | "能说说你的使用场景吗？" |
| **行为** | "你上次怎么完成的？" |
| **困难** | "哪里让你感到困惑？" |
| **感受** | "那个时刻你感觉怎么样？" |

### 追问技巧

| 技巧 | 示例 |
|------|------|
| **具体化** | "能具体说说吗？" |
| **举例** | "能举个例子吗？" |
| **对比** | "和之前的方案相比呢？" |
| **沉默** | 不说话，让用户继续 |

::: tip 不要打断

用户说一半时，不要急着回应或给建议。让用户完整表达，沉默会促使他们继续说下去。

:::

---

## 观察比询问更有价值

让用户演示使用过程，观察真实行为。

### 观察要点

| 观察 | 说明 |
|------|------|
| **犹豫停顿** | 用户不确定的地方 |
| **反复尝试** | 功能不直观 |
| **表情变化** | 困惑、沮丧、惊喜 |
| **跳过步骤** | 可能没看到某些功能 |
| **快捷键** | 高频操作 |

### 演示法请求

| 请求方式 | 示例 |
|---------|------|
| **回顾式** | "能展示一下你上次怎么用的吗？" |
| **任务式** | "请你用这个功能完成 X 任务" |
| **探索式** | "你想看看什么功能都可以" |

---

## 访谈数量

不需要大量访谈，5-10 次就能发现模式。

| 访谈数 | 价值 |
|--------|------|
| 1-3 次 | 开始发现模式 |
| 5-10 次 | 大部分问题会重复出现 |
| 10+ 次 | 新发现递减 |

::: tip 停止的信号

当第 N 个访谈没有带来新的洞察时，就可以停止了。通常 5-10 次足够。

:::

---

## 分析访谈结果

访谈后需要提炼有价值的洞察。

### 亲和图法

1. 将每条洞察写在便签上
2. 分类相关的洞察
3. 为每个分类命名主题
4. 识别最重要的主题

### 洞察提炼

| 层次 | 示例 |
|------|------|
| **原始反馈** | "找不到设置在哪里" |
| **问题** | 设置入口不明显 |
| **洞察** | 用户期望在右上角找到设置 |
| **行动** | 移动设置按钮到右上角 |

## 常见误区

| 误区 | 问题 | 正确做法 |
|------|------|---------|
| **推销产品** | 用户会说好听的话 | 保持中立，不推销 |
| **引导答案** | 用户只会附和 | 让用户自由表达 |
| **只听好的** | 错过真正的问题 | 关注负面反馈 |
| **过早解决** | 没理解就提方案 | 先理解再解决 |
| **忽略沉默** | 错过重要信息 | 让沉默继续 |

## 常见问题

### Q1: 访谈时用户不说话怎么办？

可能是问题太开放。用更具体的问题引导，或者让用户演示操作过程。

### Q2: 用户说的和做的不一致怎么办？

相信行为，不相信语言。用户说"很好用"但使用时一直皱眉，说明有问题。

### Q3: 访谈要多久？

30-45 分钟最佳。太短无法深入了解，太长用户会疲劳。

### Q4: 需要付报酬吗？

可以，但不是必须。感谢信、小礼物、早期访问权限都是回报方式。

## 本节核心要点

- ✅ The Mom Test 关注过去和具体，而非未来和笼统
- ✅ 好问题基于真实行为，坏问题询问未来意愿
- ✅ 观察用户演示比单纯询问更有价值
- ✅ 5-10 次访谈就能发现模式
- ✅ 访谈后及时整理和提炼洞察
- ✅ 避免推销、引导、过早解决等误区

用户访谈提供定性数据，接下来结合定量数据做决策。
